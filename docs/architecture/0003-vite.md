# ADR-0003: Vite als Build- und Dev-Tool

**Status:** Accepted
**Date:** 2026-05-27
**Slice:** cross-cutting

**Category:** Tech Stack

## Context

V3 braucht einen Build-Prozess der TypeScript kompiliert, JSX transformiert,
ein PWA-Bundle erzeugt und einen Dev-Server mit HMR bereitstellt. Das
`V3_CONCEPT_BRIEF.md §1` legt Vite explizit fest.

> *Diese Entscheidung war im Concept Brief explizit festgelegt, ohne
> dass Alternativen geprüft wurden — Vite ist der de-facto Standard für
> Preact + TypeScript + PWA Projekte.*

## Decision

Vite (aktuell v8) wird als Build-Tool und Dev-Server verwendet. Konfiguration in
`v3/vite.config.ts`. Der JSX-Transform ist auf `preact/jsx-runtime` gesetzt.
`vite-plugin-pwa` generiert den Service Worker (ADR-0005).

## Consequences

**Positiv:**
- HMR ist schnell (~ms statt Sekunden): gute DX beim iterativen Entwickeln.
- TypeScript wird nativ transformiert (keine extra `tsc`-Watch-Pipeline nötig).
- Plugin-Ökosystem: `vite-plugin-pwa` ist gut integriert.
- `npm run build` produziert ein produktionsreifes Bundle in ~90 ms (gemessen).

**Negativ / Trade-offs:**
- Vite nutzt esbuild für Transforms (nicht tsc für Type-Checking). Type-Fehler
  im Dev-Server werden nicht immer sofort angezeigt — `npm run build` ruft `tsc -b`
  auf und ist der authoritative Type-Check.

## Alternatives Considered

**Parcel:** Weniger Konfiguration, aber schlechterer Plugin-Support für PWA.

**webpack:** Reif und flexible, aber langsamer Dev-Server und mehr Config-Overhead.
Kein Vorteil für diesen Use-Case.
